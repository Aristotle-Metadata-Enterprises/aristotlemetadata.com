Error 404 - Page not found!

We can't find the page you've requested

Click here to go back to the home page or try searching for the page you are looking for:

But before you go... why did we decide that a missing page was an error "404"?

In a word - Metadata!

404 isn't just a "number" - its a code, from a set of well-defined and standardised codelist that all web browsers know and understand.

This metadata can then be visualised, like in the example on the right - so it can be easily digested by users like you.

Because the metadata about these codes has been recorded and endorsed, all web browsers (and users like you) can interpret this code accurately, and can make intelligent decisions about how to handle a web page.

This is what the Aristotle Metadata Registry can do for you - record your codes and definitions, so your data makes sense.

And because you know that 404 means the page isn't here, you can

  • ... check the address bar to see if you typed it correctly
  • ... remove this link from your bookmarks if you previously saved it
  • ... or use the form below to tell us about a page you wanted to find!

Web Request: Response status, IANA Code
(Data Element)
The status of a message sent from a browser to a server, using the IANA endorsed codes for HTTP.
Web Request: Response status
(Data Element Concept)
The status of a message from from a server to a browser, in response to a request for a web page. This status can be returned as a code or as a plain text message that describes the status of the request.
Web Request
(Object Class)
A computer message sent from a browser to a server to retrieve a web page (or other web resource).
Response Status
(Property)
Information from a server that describes how a browser should interpret the response message.
IANA HTTP Status Codes
(Value Domain)
A list of codes standardised by the IANA that defines specific web responses statuses.

Source: https://tools.ietf.org/html/rfc7231
