Edit content in the browser and see changes as they happen

The Aristotle Metadata Registry makes creating and editing metadata easier than before by allowing users to edit metadata using web based rich text editors. These are available to all users with the appropriate permissions without leaving the page, and full audit tracking and version control is maintained automatically to make sure administrators have full oversight of the creation and editng process.

  • Rich text editors powered by the common CKeditor.
  • AJAX Autocomplete widgets to make linking content seemless.
  • Integrated help to guide users through the edit process.

Users also have access to a number of creation wizards that guide users through the creation of new metadata. These feature all of the power and flexibility of a browser-based editor, but also guide users to similar content to make sure that they are reusing metadata where appropriate instead of duplicating content and wasting effort.

Give control to your users

Aristotle helps users keep track of their changes and manage their workload through an intuitive dashboard. From this single point of entry users can:

  • Manage their profile and details.
  • Track recent changes to metadata items and discussions.
  • View and manage permissions for themselves and teams they administer.
  • Manage additional tasks provided by extensions, such as concurrent sessions and folders of favourited content.

Users also have access to a number of quick tasks through the menu bar on every page that allows them to quickly search for or create content, access their tracked metadata items, or quickly jump to common tasks.

Powerful search and browse functionality

To help users accurately find information quickly Aristotle Metadata Registry comes with a built in search engine. By using a predictive search engine users can find and narrow down results to make reusing and understanding information easy. Search features available to users include:

  • Intelligent and predictive spelling correction based on the context of your metadata.
  • Facetting and filtering to help users narrow down to relevant results.
  • Keywords for advanced users to quickly find what they need.
All of this adds up to a seemless user experience that helps users find what they want fast.

Under the hood, this search experience is powered by the Haystack search framework which gives administrators the power to customise the search engine based on their requirements - from a light and local Whoosh engine all the way up to an Amazon-cloud, enterprise-ready ElasticSearch engine, or any combination in between.

Making collaboration a part of data management

For registered users, the Aristotle Metadata Registry comes with a powerful set of collaborative features to help teams quickly and accurately get on with the business of building knowledge. The center of this is a workgroup system that help users track their tasks and managers track roles for members. Each metadata item is kept in a contained sandbox until it is ready to be published and registred.

As metadata is progressed through these standard workflows, users can track and discuss items, seeing notifications when items they are tracking change and discussing changes with members of their team.

Build on a standard, make something new

The Aristotle Metadata Registry implements the ISO/IEC 11179 specification at its core, which together provide the flexibility of a allow for a rich extension framework. Built in is the ability for agencies to define additional fields for metadata (in ISO/IEC 11179 these are called Slots) to customise metadata for maximum coverage while still maintaining standards compliance.

When this isn't enough, whole new metadata items can be created to cover new domains - already there are open-source extensions for dataset specifications and health indicators. which gives agencies the ability to create new metadata types with a minimum of effort. The code in the above example is all thats needed to build a new metadata type, and the power of Aristotle builds all of this for free:

  • Rich-text editing screens and creation wizards.
  • Autocomplete relationships to existing related metadata.
  • Paginated browsing screens.
  • Search indexing and facets.
  • Database schemas and version tracking.

Once registered in the system all of these are automatically generated, but everything can be tailored even further to meet your specific metadata needs.

The system also comes with a download API for developers to ensure that content cna be extracted from the system in a variety of ways. Already there are extensions to support the widely recognised Data Documentation Initiative XML format and the DHIS2 metadata framework.
